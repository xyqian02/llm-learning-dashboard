# LLM Learning Dashboard — 测试方案

> ⚠️ **核心原则**：每个阶段完成后必须通过对应测试关卡（Gate），全部通过后方可进入下一阶段。若 Gate 未通过，必须修复后重新测试，不得跳过。
>
> 🔄 **Git 工作流**：每个 Gate 通过后，自动执行 `git add → commit → push`，将阶段性成果提交到 GitHub 仓库。每次提交信息格式：`gate(N): <阶段名称> — 测试通过`。

---

## 一、测试基础设施

### 1.1 测试环境

| 项目 | 说明 |
|------|------|
| Python 环境 | Anaconda base (Python 3.11.5)，路径 `/c/ProgramData/anaconda3/python` |
| 激活命令 | `source /c/ProgramData/anaconda3/etc/profile.d/conda.sh && conda activate base` |
| 后端测试框架 | pytest + httpx (FastAPI TestClient) |
| 前端开发服务器 | Vite dev server (`http://localhost:5173`) |
| 后端开发服务器 | uvicorn (`http://localhost:8000`) |
| 浏览器 | Chrome / Edge（用于手动功能验证） |
| 版本控制 | Git + GitHub |
| Git 工作流 | 每阶段测试通过 → `git add -A` → `git commit -m "gate(N): ..."` → `git push origin main` |

### 1.3 测试数据库

- 使用 SQLite 内存数据库（`:memory:`）或临时文件数据库
- 每个测试模块使用独立的 fixture，测试间数据隔离
- `conftest.py` 提供统一的 `client`（TestClient）和 `db_session` fixture

## 三、测试执行流程

```
阶段 N 开发完成
    │
    ▼
┌─────────────────────────────┐
│  运行 Gate N 测试关卡        │
│  - 自动化测试 (pytest)       │
│  - 手动功能验证              │
│  - 放行检查清单逐项打勾       │
└──────────┬──────────────────┘
           │
     ┌─────┴─────┐
     │ 全部通过？  │
     └─────┬─────┘
      是 ↙     ↘ 否
       │        修复问题
       │        重新测试
       ▼
┌─────────────────────────────┐
│  Git 提交与推送               │
│  git add -A                  │
│  git commit -m "gate(N): ..."│
│  git push origin main        │
└──────────┬──────────────────┘
           │
           ▼
     进入阶段 N+1
```

### Git 提交信息规范

| Gate | 提交信息 |
|------|----------|
| Gate 0 | `init: 项目初始化 — Git/GitHub 仓库就绪` |
| Gate 1 | `gate(1): 项目脚手架 — 前后端项目搭建完成` |
| Gate 2 | `gate(2): 数据库与种子数据 — 8表创建 + md数据导入` |
| Gate 3 | `gate(3): 后端 API — 全部路由 + 导入导出服务` |
| Gate 4 | `gate(4): 主界面与仪表盘 — 入口页 + 统计面板` |
| Gate 5 | `gate(5): 学习路线模块 — 树形交互 + 进度跟踪` |
| Gate 6 | `gate(6): 笔记模块 — Markdown编辑 + 节点关联` |
| Gate 7 | `gate(7): 收藏模块 — 四类型资源管理` |
| Gate 8 | `gate(8): 设置模块 — 备份恢复 + 导入导出` |
| Gate 9 | `gate(9): 动效与UI润色 — 动画 + 响应式 + 边界状态` |
| Gate 10 | `gate(10): 最终验收 — 全量测试通过 + 文档完整` |

### 并行阶段的测试协调

当阶段 4、5、6、7 并行开发时：
1. 各模块 Agent 先独立完成自身模块
2. 分别运行对应 Gate 的手动功能测试
3. 全部模块通过后，运行 **跨模块联动测试**（笔记↔路线、收藏↔路线跳转）
4. 联动测试通过后，Gate 4-7 整体放行，进入阶段 9

### 回归测试

阶段 9（动效打磨）完成后，在进入 Gate 10 之前，必须重跑 **Gate 3 全量 pytest**，确保 UI 改动未破坏后端 API。
