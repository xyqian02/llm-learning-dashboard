# LLM Learning Dashboard — 项目方案大纲

> ⚠️ **重要说明**：本方案为项目初始大纲，在实施过程中各模块的技术选型、数据模型、页面布局、功能细节等均可根据实际需要灵活调整。方案中的任务拆分和并行策略为推荐方案，执行时可根据资源情况动态调整任务划分。本方案应被视为"活文档"，而非僵化的执行清单。

## 一、项目概述

将 `LLM学习路线全面指南.md` 中的大模型学习路线内容，构建为一个可交互的本地 Web 应用。支持学习路线管理、进度跟踪、笔记保存、资源收藏、统计概览等功能，UI 现代美观，架构可长期扩展。

- **用户类型**：个人单机使用
- **运行方式**：本地 Web 服务（FastAPI + React），一键启动脚本
- **设计风格**：现代仪表盘卡片布局 + Notion 式温暖亲和配色 + Tailwind CSS + 适度动效

---

## 二、技术栈

| 层 | 技术 | 用途 |
|---|------|------|
| 前端框架 | React 18 + TypeScript | 组件化开发 |
| UI | shadcn/ui + Tailwind CSS | 用户指定 |
| 状态管理 | Zustand | 全局偏好（侧边栏折叠等） |
| 数据请求 | TanStack Query | API 缓存与自动刷新 |
| 路由 | React Router v6 | 多页面导航 |
| 动效 | Framer Motion | 页面过渡、列表动画、数字滚动 |
| 图表 | Recharts | 仪表盘统计图表 |
| Markdown 编辑 | @uiw/react-md-editor | 笔记编辑与预览 |
| Markdown 渲染 | react-markdown + remark-gfm | 笔记渲染 |
| 后端框架 | FastAPI (Python 3.11) | RESTful API |
| ORM | SQLAlchemy 2.0 | 数据模型与查询 |
| 数据库 | SQLite | 零配置嵌入式 |
| 数据校验 | Pydantic v2 | FastAPI 请求/响应模型 |

## 五、页面结构与路由

### 5.1 双层导航结构

```
🏠 主界面 (/)  → 全屏入口页
    ↓ 点击进入
🔲 工作台壳 (侧边栏 + 内容区)
    ├── 📊 仪表盘    (/dashboard)
    ├── 🗺️ 学习路线  (/roadmap)
    ├── 📝 笔记      (/notes)
    ├── ⭐ 收藏      (/bookmarks)
    └── ⚙️ 设置      (/settings)
```

### 5.2 各页面功能说明

**🏠 主界面（/）**
- 居中品牌区域：应用标题 + 一句学习箴言
- 3 张统计摘要卡片：总进度、进行中任务数、本周学习天数
- "继续学习"按钮（跳转到上次活跃的学习节点）
- 最近更新的笔记/收藏预览
- 背景微动态效果（粒子/网格）
- 入场动画：卡片渐入 + 标题逐字浮现

**📊 仪表盘（/dashboard）**
- 3 张核心指标卡片（数字递增动画）：总进度百分比、进行中节点数、最近学习天数
- 各阶段完成率横向条形图
- 最近活动时间线（笔记创建、收藏添加、节点完成）
- "继续学习"快捷按钮

**🗺️ 学习路线（/roadmap）**
- 分屏布局：左侧树形导航 + 右侧节点详情面板
- 树节点：展开/收起动画、状态图标（○/◐/✓）
- 详情面板：节点描述、状态切换下拉、关联笔记列表、关联资源列表
- 操作：新增子节点、编辑、删除、搜索
- 右键上下文菜单（编辑/删除/添加子节点）
- 拖拽排序（可选，MVP 后用 sort_order 手动调）

**📝 笔记（/notes）**
- 分屏布局：左侧笔记列表 + 右侧 Markdown 编辑/预览
- 列表：搜索框 + 标签筛选 + 排序（时间/标题）
- 编辑器：工具栏 + 分屏编辑/预览或切换模式
- 底部"关联学习节点"区域：弹窗树选择器多选
- 新建/编辑/删除笔记

**⭐ 收藏（/bookmarks）**
- 顶部类型 Tab 切换：论文 / 视频 / GitHub / 书籍
- 卡片网格布局，每张卡片展示类型特定字段
- 搜索 + 标签筛选
- 每张卡片显示关联学习节点标签
- 点击卡片 → 弹窗查看/编辑详情
- 论文卡片有阅读状态徽标（未读/一刷/二刷/精读）

**⚙️ 设置（/settings）**
- 数据备份：一键导出 `backup.json`，一键导入恢复
- 导入学习路线：选择 `.md` 文件，按约定格式解析
- 导出学习路线：思维导图文本格式（`├──` `└──` 缩进树）
- 导出笔记：单篇或批量导出为 `.md`
- 导出收藏：按类型导出为 `.md` 表格

## 六、动效与交互设计

### 6.1 动效方案

| 位置 | 效果 | 方式 |
|------|------|------|
| 主界面入场 | 卡片从下方渐入 + 标题逐字浮现 | Framer Motion `staggerChildren` |
| 页面切换 | 内容区 fade + 上移 10px | `AnimatePresence` + `motion.div` |
| 进度勾选 | 节点闪烁 + 进度条弹簧动画 | `layoutAnimation` + `useSpring` |
| 侧边栏 | 图标 hover 放大 + 文字滑入/出 | Tailwind + CSS transition |
| 卡片 hover | 上浮 4px + 阴影加深 | `hover:-translate-y-1` |
| 统计数字 | 从 0 递增到目标值 | `useMotionValue` + `useSpring` |
| 路线树展开 | 子节点折叠/展开弹簧动画 | Framer Motion `layout` |
| 节点状态切换 | 图标旋转 + 颜色过渡 | CSS transition |

### 6.2 动效原则
- 持续时间 200-400ms
- 使用 spring 物理曲线，自然不生硬
- 遵循 `prefers-reduced-motion` 可关闭
- 动效用于增强反馈，不阻碍操作

## 八、导入/导出格式说明

### 8.1 导入学习路线（.md 格式约定）

解析现有 `LLM学习路线全面指南.md` 中的树形结构：
- `#` 一级标题 → 根节点（文档标题）
- `##` 二级标题 → 阶段节点
- `###` 三级标题 → 主题节点
- `####` 四级标题 → 叶子节点
- 列表项 `-` 下的内容 → 节点描述/要点

扩展约定（用户后续自行编写导入时遵循）：
```markdown
## 一、阶段名称
### 1. 主题名称
#### 子主题名称
- 要点1
- 要点2
```

### 8.2 导出学习路线（思维导图文本格式）

```markdown
# 🗺️ LLM 学习路线
├── 一、认知脱盲期
│   ├── 1. Transformer 架构
│   │   ├── Self-Attention 机制
│   │   ├── 多头注意力
│   │   └── 位置编码
│   └── 2. Tokenization
└── 二、预训练
    └── ...
```

### 8.3 导出笔记（.md 格式）

```markdown
# 笔记标题
> 创建于 2025-07-15 | 关联节点: Self-Attention, Transformer

笔记正文内容...
```

### 8.4 导出收藏（.md 表格格式）

与原始 md 文件中的论文清单/GitHub 项目表格格式一致。

## 九、任务拆分与并行实施策略

### 依赖关系总览

```
阶段 0：Git 仓库初始化
  └─→ 阶段 1：脚手架（前后端同时搭建）
        ├─→ 阶段 2：数据库 + 种子数据（依赖阶段 1）
        │     └─→ 阶段 3：后端 API（依赖阶段 2）
        │           ├─→ 阶段 4：主界面 + 仪表盘（可与阶段 3 部分并行）──┐
        │           ├─→ 阶段 5：学习路线模块（依赖阶段 3）              │
        │           ├─→ 阶段 6：笔记模块（依赖阶段 3）                  ├─ 三大功能模块可并行
        │           └─→ 阶段 7：收藏模块（依赖阶段 3）                  │
        │                                                              │
        └─→ 阶段 8：设置模块（依赖阶段 3）──────────────────────────────┘
              └─→ 阶段 9：动效打磨 + UI 润色（依赖阶段 4-7）
                    └─→ 阶段 10：测试 + 文档（依赖全部功能）

🔄 每个阶段末尾：Gate 测试通过 → git commit + push
```

### 并行策略说明

本项目可采用 **单 Agent 串行执行** 或 **多 Agent 并行执行** 两种模式。以下按「任务单元」组织，每个单元标注了依赖关系和可并行范围。

---

### 阶段 0：Git 仓库初始化（阶段 1 开始前）

> 在编写任何代码之前，先建立 Git 版本控制和 GitHub 远程仓库。确保从第一行代码就有版本追溯。

| 0.3 通过 `gh` CLI 创建 GitHub 仓库 `llm-learning-dashboard` | Agent: Git | 0.1 | 5min |
| 0.4 关联远程仓库 `git remote add origin` | Agent: Git | 0.3 | 2min |
| 0.5 初始提交：`git add -A && git commit -m "init: 项目初始化" && git push -u origin main` | Agent: Git | 0.2, 0.4 | 3min |

**验证**：GitHub 仓库页面可访问，包含项目初始文件。

### 阶段 1：项目脚手架（串行基础）

> ⚠️ 此阶段为全局基础，必须最先完成，但内部前后端可并行。

| 任务 | 负责 | 依赖 | 预计耗时 |
|------|------|------|----------|

| 1.2 搭建基础布局组件：主界面壳 + 侧边栏壳 + React Router 路由框架 + Zustand store 骨架 | Agent: 前端脚手架 | 1.1 | 30min |

## 十二、Git 工作流规范

### 12.1 仓库信息
- **GitHub 仓库名**：`llm-learning-dashboard`
- **默认分支**：`main`
- **提交策略**：测试门控提交（Gate-driven commit），每个阶段测试通过后立即提交推送

### 12.2 提交信息规范
每次提交使用统一格式：`gate(N): <阶段名称> — <简要描述>`

| 阶段 | 提交信息 |
|------|----------|
| Gate 0 | `init: 项目初始化 — Git/GitHub 仓库就绪` |
| Gate 1 | `gate(1): 项目脚手架 — 前后端项目搭建完成` |
| Gate 2 | `gate(2): 数据库与种子数据 — 8表创建 + md数据导入` |
| Gate 3 | `gate(3): 后端 API — 全部路由 + 导入导出服务` |
| Gate 4 | `gate(4): 主界面与仪表盘 — 入口页 + 统计面板` |
| Gate 5 | `gate(5): 学习路线模块 — 树形交互 + 进度跟踪` |
| Gate 6 | `gate(6): 笔记模块 — Markdown编辑 + 节点关联` |
| Gate 7 | `gate(7): 收藏模块 — 四类型资源管理` |
| Gate 8 | `gate(8): 设置模块 — 备份恢复 + 导入导出` |
| Gate 9 | `gate(9): 动效与UI润色 — 动画 + 响应式 + 边界状态` |
| Gate 10 | `gate(10): 最终验收 — 全量测试通过 + 文档完整` |

### 12.3 工作流规则
1. **禁止跳过测试提交**：任何情况下不得在 Gate 测试未通过时提交代码
2. **提交粒度**：每个 Gate 一次提交，不拆分。若阶段内修复了 Bug，修复后重测通过再提交
3. **推送时机**：每次 commit 后立即 push，确保 GitHub 仓库始终与本地一致
4. **不提交的内容**：`backend/data/*.db`（数据库文件）、`node_modules/`、`__pycache__/`、`.env` 等已在 `.gitignore` 中排除
5. **回滚策略**：若某阶段出现严重问题，通过 `git revert` 回滚该阶段的提交，修复后重新走 Gate 测试
